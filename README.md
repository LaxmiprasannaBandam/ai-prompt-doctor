# 🏥 Prompt Doctor

**A prompt engineering skills lab** — an interactive Streamlit application that teaches prompt engineering through escalating challenges, evaluated by an LLM-as-judge.

> **Learn by doing.** Write prompts, get graded by an AI examiner, revise, and level up — just like a video game for prompt engineering.

---

## ✨ Features

- **5 Progressive Levels** — Start with basic role+instruction prompts, advance through structured output, few-shot learning, chain-of-thought reasoning, and adversarial defense.
- **5 Domain Tracks** — Choose from Healthcare, Customer Support, Legal, Code Review, or Finance — each with level-specific tasks.
- **LLM-as-Judge** — An AI examiner evaluates your prompt against level-specific principles, providing detailed feedback, pinpointed weaknesses, and coaching questions.
- **Live Output** — See what your prompt actually produces when run against the sample input.
- **Revision Loop** — Get feedback, revise your prompt, and resubmit until you pass.
- **Powered by OpenRouter** — Uses OpenRouter API to run student prompts and the examiner judge.

## 🎮 Gameplay

| Level | Name | Description |
|-------|------|-------------|
| 1 | **Basic** | Write a prompt with a clear role + complete instruction |
| 2 | **Structured** | Produce output in a strict JSON schema |
| 3 | **Few-Shot** | Include worked examples to handle ambiguous cases |
| 4 | **Chain-of-Thought** | Elicit step-by-step reasoning with multiple constraints |
| 5 | **Guardrails** | Handle adversarial, manipulative, or out-of-scope inputs |

## 🚀 Getting Started

### Prerequisites

- Python 3.10+
- An [OpenRouter API key](https://openrouter.ai/keys) (free tier available)

### Installation

1. **Clone or download the repository**
2. **Create and activate a virtual environment**

   ```bash
   python -m venv .venv
   .venv\Scripts\activate   # Windows
   source .venv/bin/activate  # macOS/Linux
   ```

3. **Install dependencies**

   ```bash
   pip install -r requirements.txt
   ```

   > If `requirements.txt` doesn't exist, install manually:
   > ```bash
   > pip install streamlit openai python-dotenv
   > ```

4. **Configure your API key**

   - Get your free API key at [openrouter.ai/keys](https://openrouter.ai/keys)
   - Edit the `.env` file and replace the placeholder:
     ```
     OPENROUTER_API_KEY=sk-or-v1-...
     ```
   - Alternatively, enter your key in the app's **API Key Settings** section in the sidebar.

### Running the App

```bash
streamlit run app.py
```

The app will open in your default browser at `http://localhost:8501`.

## 🎯 How to Play

1. **Select a Domain** — Choose from Healthcare, Customer Support, Legal, Code Review, or Finance.
2. **Enter Your Prompt** — Write a prompt in the editor on the left panel.
3. **Submit for Evaluation** — Click the submit button to run your prompt and get it graded.
4. **Review the Verdict** — The right panel shows:
   - ✅/❌ per principle (pass/fail)
   - 🗣️ Weakness quotes from your prompt
   - ❓ Coaching questions to guide your revision
   - 📤 Live output generated by your prompt
5. **Revise & Resubmit** — Fix the issues and try again.
6. **Advance** — Once all principles pass, advance to the next level (or reset if you've completed all 5!).

|--------|---------------|

## 🧠 How the Examiner Works

1. The **examiner** receives the student's prompt, the current level definition, the sample input, and the live output.
2. It grades each principle (role, instruction, output_format, few_shot, chain_of_thought, guardrails) individually.
3. For each **failed** principle, it quotes the exact weak phrase and asks a coaching question.
4. It outputs a structured JSON verdict — no extra text, no corrected prompts.
5. The verdict is **"pass"** only if **every** principle passes.

> The examiner **never writes a corrected version** of your prompt — it diagnoses, coaches, and guides you toward the fix yourself. That's how you learn!

## 🌟 Pro Tips

- **Be specific** — Vague instructions produce vague results. Be explicit about what you want.
- **Use the feedback** — The weakness quotes and coaching questions are designed to help you improve.
- **Test edge cases** — Especially in Levels 3–5, think about what could go wrong and handle it.
- **Iterate** — The revision loop is where the learning happens. Don't be afraid to fail and try again.
- **Switch domains** — Try different domains to see how prompt engineering principles apply across contexts.
